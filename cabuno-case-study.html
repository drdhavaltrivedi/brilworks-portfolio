Project Overview

Cabuno is an on-demand mobility application designed to bridge the gap between urban travelers and reliable drivers. Built as a cross-platform solution using Expo, it supports Android, iOS, and Web platforms with a unified codebase.

The Problem

Commuters in urban environments often struggle with uncertain ride availability, inconsistent pickup experiences, and a lack of transparency regarding fares and ride progress. Traditional systems often lack cohesive real-time communication between the rider and driver.

The Solution

We developed a location-aware matching engine that utilizes Firebase's real-time capabilities to provide live status updates, transparent routing, and a frictionless booking journey from authentication to completion.

04

Challenges Solved

Stale Ride Data

Status changes can happen in milliseconds. We solved this by combining Firestore's real-time snapshots with React Query's cache invalidation, ensuring the UI always reflects the database state without manual refreshes.

Booking Conflicts

Preventing duplicate active rides was critical. We implemented a rigorous lifecycle guard that checks for ongoing trip states before allowing a user to enter a new booking flow.

Reliable Session Cleanup

Logging out isn't just about clearing a token; it's about security. We built a structured sign-out pipeline that clears FCM notification tokens and resets all global stores to prevent data leakage.
